üîß How to Fix This

  1. Go to Google Cloud Console: console.cloud.google.com
  2. Select your project (or create a new one)
  3. Enable Billing:
    • Go to "Billing" in the left menu
    • Link a billing account (Google gives $200 free credit)
    • ‚ö†Ô∏è Without billing, maps won't load!
  4. Enable Required APIs:
    • Go to "APIs & Services" ‚Üí "Library"
    • Search and enable each:
      • ‚úÖ Maps JavaScript API
      • ‚úÖ Places API
      • ‚úÖ Directions API
      • ‚úÖ Geocoding API
  5. Create/Update API Key:
    • Go to "APIs & Services" ‚Üí "Credentials"
    • Click "Create Credentials" ‚Üí "API Key"
    • Copy the API key
  6. Configure API Key Restrictions (Optional but Recommended):
    • Click on your API key to edit
    • Under "Application restrictions":
      • Select "HTTP referrers (web sites)"
      • Add: http://localhost:*/*
      • Add: http://127.0.0.1:*/*
      • Add your production domain when ready
    • Under "API restrictions":
      • Select "Restrict key"
      • Select: Maps JavaScript API, Places API, Directions API, Geocoding API
    • Click "Save"
  7. Update TripSync .env file:
    • Open: frontend/.env
    • Update: VITE_GOOGLE_MAPS_API_KEY=your_new_api_key
    • Restart frontend: npm run dev
  8. Wait 5 minutes for API key changes to propagate
  9. Refresh this page and test again
